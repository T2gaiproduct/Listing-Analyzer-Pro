# Prompt for Replit – Credit Based Billing System + Package Management

You are working on an existing SaaS web application project.  
Implement a complete **Credit-Based Billing & Usage System** inspired by the Upwork “Buy Connects” model.

The system should allow customers to:
1. Subscribe to monthly packages
2. Receive monthly credits based on package
3. Buy additional credits anytime
4. Track credit usage in real time
5. View transparent deduction rules
6. Allow admin to fully manage packages and credits dynamically

---

## 1. CREDIT SYSTEM

Create a universal credit system.

### Credit Value
- 1 Credit = USD $0.10

### Additional Credit Purchase
Customers should be able to purchase credits anytime.

Example:
- 50 Credits = $5
- 100 Credits = $10
- 200 Credits = $20
- 500 Credits = $50

Also allow:
- Custom credit amount input
- Auto price calculation
- Stripe/PayPal integration ready structure

Recreate package structure exactly like below.

## Packages

| Package | Products | Monthly Credits | Monthly Price |
|---|---|---|---|

| Growth | 5 Products | 225 Credits | $49 |
| Pro | 10 Products | 450 Credits | $99 |
| Enterprise | 25 Products | 1125 Credits | $249 |

### Annual Pricing
Show yearly billing with 20% discount.

Examples:
- Growth → $470/year
- Pro → $950/year
- Enterprise → $2390/year

Display this deduction table everywhere:
